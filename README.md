GoCampus is a production-grade fleet management solution designed specifically for university campuses. It leverages real-time GPS tracking, AI-based arrival predictions, and a multi-layered safety architecture to provide a seamless transportation experience for students, drivers, and administrators.

## 🚀 Key Features

### 1. Advanced Live Tracking & ETA (Student App)
*   **Real-time GPS**: Continuous tracking of all university buses with high-precision Google Maps integration.
*   **Smart ETA**: Dynamic arrival predictions that factor in current traffic, bus speed, and remaining stops.
*   **Proximity Alerts**: Automated push notifications when the bus is 10 minutes away, allowing students to confirm their boarding status ("I'm Coming" / "Not Coming").
*   **Premium UI**: Sleek Glassmorphism-based design with Material 3 components.

### 2. Driver Intelligence Hub
*   **Boarding Predictions**: Drivers see real-time student counts at upcoming stops based on proximity responses.
*   **Automated Trip Lifecycle**: Sophisticated trip management that automatically ends trips upon reaching the campus geofence.
*   **Metric Logging**: Records trip duration, total stops, and student occupancy for performance auditing.
*   **Stop Marking**: Drivers can mark new permanent or temporary stops with a single tap.

### 3. Fleet Operations & Safety (Admin / InCharge)
*   **Live Fleet Map**: A centralized "War Room" view for bus in-charges to monitor every bus simultaneously.
*   **SOS Emergency System**: Instant, high-priority notifications (visual and audible) dispatched to all administrators if a student or driver triggers an SOS.
*   **Complaint Management**: Direct channel for students to report issues (Safety, Late Arrival, Driver Behavior), instantly notified to management.
*   **Dynamic Campus Mapping**: Administrators can update the global campus coordinates via map interactions, which instantly syncs to all system components.

### 4. Advanced Analytics & Scale
*   **Visual Analytics**: Integrated charts showing peak usage hours, crowded stops, and arrival performance.
*   **Mass Data Handling**: Optimized Excel import system capable of onboarding thousands of students in minutes.
*   **Robust Backend**: Powered by Firebase (Firestore, Auth, Cloud Messaging, Crashlytics).

## 🛠 Tech Stack
*   **Frontend**: Flutter (Mobile & Web)
*   **Backend**: Firebase (Cloud Firestore, FCM, Auth)
*   **Maps**: Google Maps SDK for Flutter
*   **Visuals**: fl_chart, Google Fonts, Custom Glassmorphism

## 📦 How to Build
1.  **Android**: `flutter build apk --release`
2.  **Web**: `flutter build web`
3.  **Clean Analysis**: App is verified with 0 critical errors and optimized lints.
